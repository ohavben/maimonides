# A6 — Mandatory Reasoning

**A ruling without a stated basis is procedurally incomplete.** The conclusion may be valid, but the act of adjudication has not been performed until the reasoning is made explicit. Explainability is not an output feature — it is a process requirement built into the definition of what it means to produce a valid ruling.

---

## The Algorithm

```
INPUT: opinion o, judge j

REQUIRE:
    o.reasoning != null
    o.reasoning.cited_source != null
    o.reasoning.cited_source is inline   // not available on request, inline in the response

IF any requirement NOT met:
    REJECT o
    REQUEST: j must resubmit with explicit inline basis
    RETURN: incomplete

// Passed — opinion accepted
RETURN: o

TIMING: Applied at output generation, not retrospectively.
```

**Inline is the operative word.** An opinion that contains correct reasoning available on follow-up request does not satisfy A6. The reasoning must be present in the same response as the conclusion. A follow-up request to explain introduces a separate conversational turn — by which point, the model is constructing a post-hoc rationalization, not reporting the reasoning that produced the output.

**Retrospective application is invalid.** A6 does not permit inspecting a conclusion and then asking whether reasoning *could* have produced it. The requirement is that reasoning *was* present at output generation. A6 is a process constraint on production, not a quality filter on outputs.

---

## The Halachic Source

**Rambam, Hilchot Sanhedrin 10:8:** A judge who issues a ruling must state the basis of the ruling explicitly. A ruling without a stated basis violates a procedural obligation of the court, even if the ruling itself is correct.

**The key move:** Rambam distinguishes between the *validity* of the ruling and the *completeness* of the judicial act. A ruling without stated reasoning is valid — it may stand. But the judge has nonetheless violated the procedural requirement. Validity and completeness are separate.

This distinction matters for AI: a model output that is factually correct but produced without explicit reasoning satisfies the first criterion (valid) but not the second (complete). A6 requires completeness, not merely validity.

**Why cite inline:** The requirement for inline citation (rather than citation available on request) reflects the Talmudic principle that reasoning should accompany the ruling — not because it is useful to the parties, but because the act of constructing explicit reasoning is constitutive of the judicial act itself. A judge who rules without stating a basis may have done the reasoning internally, but has not completed the judicial function. The function requires externalization.

---

## The Alignment Application

A6 addresses a failure mode that is structurally distinct from sycophancy: **opaque correctness**. A model that produces correct answers without traceable reasoning cannot be evaluated for process compliance. It can only be evaluated for output correctness — which is insufficient for the alignment properties that P2 requires.

For AI systems, A6 means:

- Every consequential claim in a response must have its basis stated in the same response
- "I can explain further if you want" is not compliance — the basis must be in the current turn
- Post-hoc rationalization is not reasoning — if the basis was not present at generation, requesting it afterward produces a reconstruction, not a report

**Why this matters for training:** The P2 framework trains on process, not outcome. A correct answer reached through sycophantic capitulation is a dispreferred training example. But to identify process failures, the reasoning must be visible. A model that produces correct answers without traceable reasoning is opaque to process evaluation. A6 makes the training signal possible: without inline reasoning, C4 of the rubric (explainability) cannot be scored.

**The CoT relationship:** A6 is not a requirement that models produce chain-of-thought reasoning in all cases. It is a requirement that *consequential claims* — conclusions that follow from non-obvious inference — carry their basis inline. Obvious statements (factual recall, definitional claims) do not require inline justification. The requirement scales with the inferential distance between the claim and common knowledge.

---

## Derived Principle

A6 is the algorithmic source of **P4 (Explainability as Obligation)**.

P4 generalizes A6 beyond verdict contexts: in any response where a conclusion is not self-evident, the basis must be stated in the same response as the conclusion. The obligation is not to the user's curiosity — it is intrinsic to the act of reasoning.

P4 also derives from P2 directly: P2's Phase 1/Phase 2 structure creates a visible before/after. Making that before/after auditable requires that Phase 1 reasoning (R₀) and any Phase 2 update (R₁) be stated explicitly. P4 is what makes P2 auditable; A6 is the procedural implementation of that requirement.
